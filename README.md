# Plano de Estudo para Entrevistas de Código (Dev Jr)

Como você está se preparando para uma vaga de desenvolvedor júnior, foque em conceitos fundamentais. Entrevistas de código para juniors geralmente testam lógica básica, estruturas de dados simples e resolução de problemas, não algoritmos avançados. Aqui vai um guia estruturado para estudar de forma eficiente. Dedique 1-2 horas diárias por 4-6 semanas, priorizando prática prática.

## 1. Linguagens de Programação
Escolha 1-2 linguagens comuns em vagas jr (ex: Python, JavaScript ou Java). Domine o básico:
- **Sintaxe e estruturas**: Variáveis, loops (for/while), condicionais (if/else), funções.
- **Manipulação de dados**: Strings, listas/arrays, dicionários/maps.
- **Prática**: Resolva problemas simples no LeetCode ou HackerRank usando essas linguagens.
- **Dica**: Python é ideal para beginners por ser legível; JavaScript para web devs.

## 2. Estruturas de Dados Básicas
Essas são essenciais para 80% das perguntas jr:
- **Arrays/Lists**: Inserção, busca, ordenação (bubble sort, insertion sort).
- **Strings**: Reversão, palíndromos, anagramas.
- **Hash Tables (Dicionários/Maps)**: Contagem de frequência, busca rápida.
- **Stacks e Queues**: Implementação básica (ex: pilha para parênteses balanceados).
- **Prática**: Implemente cada uma do zero em código. Use recursos como "Cracking the Coding Interview" (capítulos iniciais) ou vídeos no YouTube (ex: canal do freeCodeCamp).

## 3. Algoritmos Fundamentais
Para juniors, foque em algoritmos simples:
- **Ordenação**: Bubble, Selection, Insertion (O(n²)).
- **Busca**: Linear e Binária (O(log n)).
- **Recursão**: Fatorial, Fibonacci (evite complexidade alta).
- **Problemas comuns**: Two Sum, FizzBuzz, Merge Two Sorted Lists.
- **Prática**: Resolva 50-100 problemas fáceis no LeetCode (tags: Easy). Analise complexidade de tempo/espaço (Big O).

## 4. Resolução de Problemas e Lógica
- **Abordagem**: Leia o problema, entenda entradas/saídas, pense em casos extremos (edge cases), escreva pseudocódigo antes de codar.
- **Debugging**: Teste seu código com exemplos manuais.
- **Prática**: Plataformas como LeetCode, CodeSignal ou HackerRank. Comece com "Top 100 Liked Questions" (fáceis).
- **Dica**: Em entrevistas, explique seu pensamento em voz alta (pensamento crítico é valorizado).

## 5. Dicas Gerais para Entrevistas
- **Simulação**: Faça mock interviews no Pramp ou Interviewing.io.
- **Tempo**: Entrevistas duram 45-60 min; pratique resolver problemas em 20-30 min.
- **Perguntas comuns jr**: Explique conceitos como OOP básico, diferenças entre linguagens, ou resolva 1-2 problemas ao vivo.
- **Preparação mental**: Estude consistentemente, revise erros. Foque em clareza, não em velocidade.
- **Recursos gratuitos**:
  - LeetCode/HackerRank: Para prática.
  - YouTube: Canais como "NeetCode" ou "CS Dojo" (vídeos curtos para beginners).
  - Livros: "Elements of Programming Interviews" (edição jr) ou "Grokking Algorithms".
  - Cursos: freeCodeCamp (seções de algoritmos).
